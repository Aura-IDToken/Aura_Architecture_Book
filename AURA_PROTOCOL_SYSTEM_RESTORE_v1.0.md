# AURA_PROTOCOL_SYSTEM_RESTORE_v1.0

## Cel

Dokument odtwarzający pełny kontekst prac architektonicznych nad
projektem Aura.

## Zakres

Analiza objęła: - aura-poc-a-core - aura-guard-v1.3 - aura-sign-mvp

## Główne odkrycia

-   Aura jest protokołem, nie pojedynczą aplikacją.
-   Implementacje są referencyjne.
-   Konstytucja pełni rolę nadrzędnej specyfikacji.
-   Największą wartością projektu jest model zaufania i zgodności.

## Architektura docelowa

Mathematical Core → Evidence Model → Decision Engine → Attestation
Engine → Cryptographic Core → Operational Core

## Najważniejsze wnioski

1.  Najpierw specyfikacja.
2.  Następnie niezmienniki.
3.  Następnie testy zgodności.
4.  Potem Conformance Core.
5.  Dopiero później zmiany w implementacji.
